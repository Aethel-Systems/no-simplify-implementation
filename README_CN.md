# No-Simplify Implementation Skill

## **没有**`//简化实现`、**没有**`//此处省略`、没有`//此处使用简单的...实际需...`！
**工业级完整性强制协议** —— 绝不接受任何形式的懒惰、占位、简化或玩具级 Demo。

## 简介

`no-simplify-implementation` 是一个专为代码生成场景设计的强约束 Skill。它强制 AI 产出**100% 完整、生产就绪、可直接上线**的代码，彻底杜绝 TODO、FIXME、...、空异常处理、空函数、Mock 实现以及任何“简化演示”痕迹。

不同于市面上大多数“代码辅助”工具，这个 Skill 内置了一套**动态 + 静态**混合的严苛分析器（包含 AST 解析、结构度量、Mock 检测启发式），能在生成后立即进行工业级完整性校验。只有通过 linter（退出码 0）才能视为合格。

### 核心理念

现有的 Skill 和项目往往标榜“简洁 + 保证健壮”，听起来很专业，实则为模型提供了更多可乘之机。那些自以为代码结构好看、口号喊得响亮（动辄“工业级别”“企业级”），实际却只是玩具级 Demo 的 Skill 项目，恰恰给了懒惰实现最大的生存空间——表面干净整洁，内里空洞无物，真正需要健壮逻辑的地方永远留着“后续补充”或一句漂亮的注释了事。这种“优雅的偷懒”才是最危险的。**我们选择反其道而行之**：宁可多写一行真实逻辑，也不接受一行看似精炼的废话。

## 特性

- **多语言懒惰检测**：支持 Python、JS/TS、Go、Java、C/C++ 等主流语言
- **上下文感知 Lexer**：区分字符串、注释与真实代码，避免误判
- **Hardcore 结构分析器**：检测空异常、空函数、常量返回、仅打印的 Mock 实现、复杂度异常低的“复杂命名”函数
- **中文/英文/符号全覆盖**：包括“此处省略”“简化实现”“for simplicity”“...” 等各类占位
- **增量修复循环**：发现问题立即定位、编辑、重验，直至通过
- **并行扫描 + 丰富报告**：支持目录批量扫描、JSON 输出、彩色终端报告

## 安装与使用（OpenClaw 环境）

```bash
clawhub install no-simplify-implementation
```
对话时手动`/no-simplify-implementation` （模型不会自动调用）

### 推荐工作流

1. 在任何需要生成代码的任务前激活本 Skill
2. 模型开始编写工业级别代码
3. 最终全项目扫描通过后再交付用户（你）

**核心 linter 命令**（在 workspace 目录下）：
```bash
python3 skills/no-simplify-implementation/scripts/main.py workspace/
# 或单个文件
python3 skills/no-simplify-implementation/scripts/main.py workspace/yourfile.py
```

## 适用场景

- 大型系统组件开发
- 生产级脚本 / 服务 / 工具链实现
- 重构遗留代码，要求彻底消除技术债
- 需要对 AI 输出进行质量把关的团队或个人

## 为什么需要它？

因为大多数 AI 生成的代码，在“简洁优雅”的包装下，实际上经不起推敲。一旦遇到真实的生产环境（边界条件、错误处理、资源管理、配置加载），就立刻露出玩具本质。

`no-simplify-implementation` 存在的意义，就是把这种“看起来像、实际上不是”的代码直接扼杀在交付之前。它不接受妥协，不接受“演示用途”，不接受“后续再完善”。它只接受**能跑、能维护、能上线**的完整实现。
